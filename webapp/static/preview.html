MVDR beamformer noise reduction · 3 messages
How does MVDR beamforming reduce noise, and is the distortionless constraint actually preserved in practice?
From your library

MVDR (Minimum Variance Distortionless Response) beamforming minimizes the output power while holding the array's response toward the look direction fixed at unity. By steering deep nulls toward interference and diffuse noise, it suppresses everything except the target, which is what reduces noise 1.

The key is the distortionless constraint wᴴd = 1: the optimal weights w = (R⁻¹d)/(dᴴR⁻¹d) are derived precisely so the desired signal passes undistorted while the noise covariance R is whitened 2.

In practice the constraint holds to numerical tolerance, but finite snapshots make the sample covariance ill-conditioned, so diagonal loading is added to keep R⁻¹ stable — at the cost of slightly shallower nulls 3.

def rk4_step(bodies, G, dt):
    # classic 4th-order Runge-Kutta for the gravitational ODE
    k1 = accel(bodies, G)
    k2 = accel(advance(bodies, k1, dt / 2), G)
    k3 = accel(advance(bodies, k2, dt / 2), G)
    k4 = accel(advance(bodies, k3, dt), G)
    return advance(bodies, (k1 + 2*k2 + 2*k3 + k4) / 6, dt)

def total_energy(bodies, G):
    ke = 0.5 * sum(b.m * b.v @ b.v for b in bodies)
    pe = -G * pair_potential(bodies)
    return ke + pe   # conserved => dE/E ~ 1e-13
